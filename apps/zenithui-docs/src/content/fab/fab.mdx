## üìç Position

Place the FAB on screen with the `position` prop on `Root`:

- **Corners:** `top-left`, `top-right`, `bottom-left`, `bottom-right`
- **Edges:** `top-center`, `bottom-center`, `center-left`, `center-right`
- **Center:** `center`

Use `xOffset` and `yOffset` (in pixels) to nudge the trigger from the edges.

---

## üìÇ Content placement

Use the `placement` prop on `Content` to open the panel relative to the trigger: `top`, `bottom`, `left`, or `right`. The component flips automatically if there isn‚Äôt enough space in the viewport.

## üß© Menu building blocks

Use inside `Fab.Content`:

- **Fab.Group** ‚Äì Wraps a set of items; use with **Fab.Label** for a section title.
- **Fab.Label** ‚Äì Section label (e.g. "Quick actions").
- **Fab.Separator** ‚Äì Visual divider between sections.
- **Fab.Item** ‚Äì Clickable menu item; closes the FAB on select by default (`closeOnSelect`).
- **Fab.CheckboxItem** ‚Äì Toggle item with `checked` / `onCheckedChange`; use **Fab.ItemIndicator** for a checkmark.
- **Fab.RadioGroup** + **Fab.RadioItem** ‚Äì Single selection; use **Fab.ItemIndicator** for the selected state.
- **Fab.ItemIndicator** ‚Äì Renders when the parent item is checked/selected; style with `group-data-[state=checked]:inline-flex` (or use the registry styles).

## üîê Accessibility

- Trigger has `aria-expanded` and `aria-haspopup="true"`.
- Content has `role="menu"` and `aria-modal="true"`.
- Items use `role="menuitem"`, `menuitemcheckbox`, or `menuitemradio` as appropriate.

---

## üß† Best practices

- Prefer outline-based focus styles (or the registry trigger) so `transform: scale()` doesn‚Äôt shift the trigger position.
- Use controlled `open` / `onOpenChange` when you need to sync with other UI (e.g. close when navigating).
